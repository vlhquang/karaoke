Generics (Kiểu dữ liệu tổng quát)

Tại sao cần Generics?

Generics cho phép các lớp, interface và phương thức hoạt động trên nhiều kiểu dữ liệu khác nhau trong khi vẫn đảm bảo an toàn kiểu (type safety) tại thời điểm biên dịch.

Không có Generics, bạn phải ép kiểu (cast) thủ công và có thể gặp lỗi ClassCastException tại runtime.

Cú pháp Generics

// Không có generics (không an toàn)
List list = new ArrayList();
list.add("hello");
String s = (String) list.get(0); // Phải ép kiểu thủ công

// Với generics (an toàn)
List<String> list = new ArrayList<>();
list.add("hello");
String s = list.get(0); // Không cần ép kiểu

Lớp Generic

Bạn có thể tạo lớp generic bằng cách sử dụng tham số kiểu (type parameter) ký hiệu bằng chữ cái in hoa (thường là T, E, K, V...):

public class Box<T> {
    private T content;

    public void set(T content) { this.content = content; }
    public T get() { return content; }
}

// Sử dụng
Box<String> stringBox = new Box<>();
stringBox.set("Hello");
IO.println(stringBox.get()); // "Hello"

Wildcards (?)

Ký tự ? đại diện cho một kiểu dữ liệu không xác định, hữu ích khi viết code tổng quát hơn:

// Cho phép List của bất kỳ kiểu nào
public void printList(List<?> list) {
    for (Object o : list) {
        IO.println(o);
    }
}
