Kế thừa trong Java

Kế thừa cho phép bạn xây dựng các lớp mới dựa trên các lớp hiện có, tái sử dụng và mở rộng hành vi của chúng.

Kế thừa là gì?

Trong Java, mỗi lớp đều được phép có một lớp cha (superclass). Lớp con kế thừa (inherit) tất cả các trường và phương thức không có phạm vi truy cập private từ lớp cha. Để kế thừa từ một lớp, ta dùng từ khóa extends: 

class MountainBike extends Bicycle {
    // MountainBike có thêm các trường và phương thức của riêng nó
    public int seatHeight;
}

Ghi đè phương thức (@Override)

Lớp con có thể ghi đè (override) một phương thức được kế thừa từ lớp cha, cung cấp một triển khai mới. Nên dùng annotation @Override để trình biên dịch kiểm tra. 

public class Animal {
    public String makeSound() {
        return "Some sound";
    }
}

public class Dog extends Animal {
    @Override
    public String makeSound() {
        return "Woof!";
    }
}

Từ khóa super

Từ khóa super cho phép lớp con:

  • Gọi constructor của lớp cha: super()
  • Truy cập phương thức của lớp cha: super.methodName()

Từ khóa final

Bạn có thể khai báo một lớp là final để ngăn không cho lớp đó bị kế thừa. Điều này thường được dùng để bảo vệ tính bất biến của một lớp (ví dụ: lớp String).
