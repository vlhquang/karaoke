Rẽ nhánh với Switch Expression

Switch Expression là gì?

Giống như câu lệnh switch truyền thống, switch expression có thể có nhiều đường dẫn thực thi. Tuy nhiên, thay vì chỉ thực hiện các câu lệnh, nó có thể trả về một giá trị.

Cú pháp mũi tên (Arrow Syntax)

Switch expression giới thiệu cú pháp L ->. Với cú pháp này, chỉ khối mã ở phía bên phải mũi tên được thực thi nếu nhãn khớp. Không có cơ chế fall-through, vì vậy bạn không cần từ khóa break.

int quarter = 1;
String quarterLabel = switch (quarter) {
    case 0 -> "Q1 - Mùa Đông";
    case 1 -> "Q2 - Mùa Xuân";
    case 2 -> "Q3 - Mùa Hạ";
    case 3 -> "Q4 - Mùa Thu";
    default -> "Quý không hợp lệ";
};

Từ khóa yield

Nếu bạn sử dụng cú pháp case L: truyền thống bên trong một switch expression, bạn phải sử dụng từ khóa yield để trả về giá trị từ khối đó.

int quarterLabel = switch (quarter) {
    case 0:
        yield "Q1 - Mùa Đông";
    case 1:
        yield "Q2 - Mùa Xuân";
    default:
        yield "Unknown";
};

Tính toàn vẹn (Exhaustiveness)

Khác với câu lệnh switch thông thường, switch expression phải xử lý tất cả các giá trị có thể có của biến lựa chọn. Thông thường, điều này có nghĩa là bạn bắt buộc phải có một nhánh default.
