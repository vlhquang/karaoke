Debugging trong Java

Debugging là gì?

Debugging (gỡ lỗi) là quá trình tìm và sửa lỗi trong chương trình. Hầu hết các IDE Java hiện đại đều tích hợp debugger mạnh mẽ giúp bạn dừng chương trình tại bất kỳ dòng nào và kiểm tra giá trị biến.

Breakpoints

Breakpoint là điểm dừng trong code. Khi debugger đạt đến breakpoint, chương trình dừng lại và bạn có thể:

  • Xem giá trị của tất cả biến tại thời điểm đó.
  • Thực thi từng dòng một (Step Over, Step Into, Step Out).
  • Thay đổi giá trị biến trực tiếp.

Debug trong IDE

  • VS Code: F5 để bắt đầu debug, F9 để đặt/xóa breakpoint.
  • IntelliJ IDEA: Shift+F9 để debug, F9 đặt breakpoint.
  • Eclipse: F11 để debug, Ctrl+Shift+B để đặt breakpoint.

Mẹo debug hiệu quả

  • Đọc kỹ stack trace khi có exception.
  • Dùng logging (IO.println() hoặc java.util.logging) để theo dõi luồng chương trình.
  • Viết unit tests để phát hiện lỗi sớm.
  • Dùng binary search để cô lập vùng code có lỗi.
