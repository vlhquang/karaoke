Virtual Threads (Luồng ảo)

Virtual Threads là gì?

Được giới thiệu chính thức trong Java 21 (Project Loom), Virtual Threads là luồng nhẹ (lightweight threads) được quản lý bởi JVM thay vì hệ điều hành. Chúng cho phép viết mã đồng thời (concurrent) với số lượng luồng hàng trăm nghìn mà không gây cạn kiệt tài nguyên.

Vấn đề với Platform Threads

Mỗi platform thread (luồng OS truyền thống) tiêu tốn ít nhất vài MB bộ nhớ. Khi một luồng chờ I/O (như gọi API, truy vấn database), nó bị block và tốn tài nguyên. Điều này giới hạn khả năng mở rộng (scalability) của ứng dụng.

Cách sử dụng Virtual Threads

// Tạo và chạy một virtual thread
Thread.ofVirtual().start(() -> {
    IO.println("Chào từ Virtual Thread!");
});

// Dùng Executor cho nhiều virtual threads
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    for (int i = 0; i < 100_000; i++) {
        int taskId = i;
        executor.submit(() -> {
            // Tác vụ I/O - sẽ không block
            IO.println("Task " + taskId);
        });
    }
}
// Executor tự đóng khi ra khỏi try block

Khi nào nên dùng?

Virtual threads phù hợp nhất cho các tác vụ I/O-bound (nhiều thời gian chờ mạng, database, file). Chúng không cải thiện các tác vụ CPU-bound (tác vụ tính toán nặng).
